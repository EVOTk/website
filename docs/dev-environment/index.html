Setup Dev Environment

Setup Backend

Download go dependencies with golang/dep.

$ dep ensure

Run golang server.

$ go run app.go

Setup UI

Commands must be executed inside the ui directory.

Download dependencies with npm.

$ npm install

Star the UI development server.

$ npm start

Open http://localhost:3000 inside your favorite browser.

The UI requires a Gotify server running on localhost:80 this can be adjusted inside the ui/src/index.tsx.

Execute Backend Tests

The tests can be executed with:

$ make test
# or
$ go test ./...

Execute UI (end2end) Tests

Build the ui because the end2end test should be run against the production build. (This needs to be done on every change in the UI)

$ (cd ui && npm run build)

Now execute the tests with npm

$ (cd ui && npm run test)

Executing Static-Checks

The following command checks the formatting and executes some linters like tslint and govet.

$ make check

Building Gotify

Build the UI

$ (cd ui && npm run build)

Run packr (embeds static assets in go binaries)

$ packr

Build the Go Binary

$ go build -o build/gotify-server

Hurray! You built Gotify. This is only a development build, see .travis.yml for settings the mode to prod.

Cross-Platform

The project has a CGO reference (because of sqlite3), therefore a GCO cross compiler is needed for compiling for other platforms. See .travis.yml on how we do that.
